# AIMarket Pro — Project Functionalities

This document outlines the core functional modules of the AIMarket Pro frontend application. It serves as a high-level guide for backend developers to understand the platform's architecture and the business logic required to support the user interface.

## 1. Core Architecture & Concept
AIMarket Pro is a **"Done-For-You AI Ad Platform"**. The primary value proposition is allowing users to create, optimize, and manage digital advertising campaigns entirely through AI. 
Instead of tweaking exact bids and complex targeting, the user gives the AI a product, and the AI generates the ad creatives, writes the copy, determines the best audience, and automatically shifts budget based on ROI down the line.

## 2. Ad Campaign Wizard (`CreateAd.jsx`)
This is the core engine of the platform where users build new ads. It revolves around a multi-step state machine:
- **Step 1: AI Analysis.** The user uploads a product image/video OR types a description. The backend AI analyzes this input to understand the product.
- **Step 2: Creative Generation.** The backend uses image generation (e.g., HuggingFace) and text APIs (e.g., Groq) to generate targeted ad copy (headlines, hooks, primary text) and ad creatives based on the analysis.
- **Step 3: Audience & Targeting.** The user selects targeting parameters (Locations, radius, demographics). The backend must support saving these targeting constraints.
- **Step 4: Platform Selection & Budget.** The user splits their daily budget among connected platforms (Facebook, Instagram, Google, YouTube). The backend must validate the budget against minimum limits.
- **Step 5: Review & Approval.** The user previews the ad. They can "Approve & Launch," causing the backend to publish the campaign to Meta/Google APIs, or they can "Request Changes," restarting parts of the flow.

## 3. Real-Time Campaign Dashboard (`Dashboard.jsx`)
The dashboard gives users a bird's-eye view of their active and historical campaigns.
- **Campaign Listings:** Shows campaign status (Active, Paused, Completed), total spend, and high-level performance numbers.
- **AI Optimization Engine:** Features an Optimization Toggle. 
  - *Auto-Optimization Mode:* The backend should have a cron job or worker that automatically adjusts bids, shifts budgets between ad sets, and pauses underperforming ads without user intervention.
  - *Manual Approval Mode:* The backend detects scaling opportunities or underperformance and queues them up as "Insights" for the user to manually approve before API changes are made to the ad networks.

## 4. Live Analytics & ROI Tracking (`Analytics.jsx`)
Deep-dive metrics for campaigns.
- **Metrics Engine:** The backend must aggregate data from Google Ads API, Meta Graph API, etc., and return unified metrics (Total Clicks, Conversions, Impressions, Total Spend, Overall ROI).
- **Timeframe Filtering:** Data must be aggregatable by day, week, month, or custom date ranges to support the frontend charts.

## 5. Lead Tracking / CRM (`CRM.jsx` & `AdClick.jsx`)
The platform acts as a mini-CRM for leads captured directly from lead-generation campaigns.
- **Ad Landing Pages (`AdClick.jsx`):** Public-facing pages generated by the system for the ads. When an end-user fills out the form, it saves the lead.
- **CRM Table (`CRM.jsx`):** Allows the advertiser to view all captured leads, sorted by campaign, with estimated values, date captured, and the specific "Audience Segment" the lead belonged to (e.g., "Tech Enthusiasts").

## 6. Integrations & Billing (`Billing.jsx`)
Manages the user's connections and payments.
- **Ad Account OAuth:** Connecting Facebook, Instagram, Google, and TikTok accounts via OAuth so the backend can manage ads on their behalf. Ad spend is charged *directly* to the user's card on those platforms.
- **Platform Service Fee:** A monthly SaaS subscription fee (e.g., via Stripe or PayPal) paying for the AIMarket Pro AI optimization tool itself. Includes fetching invoice histories.

## 7. Configuration & AI Settings (`Settings.jsx`)
Manages system-level configurations.
- **Bring-Your-Own-Key (BYOK):** The user can provide their own Groq API keys and HuggingFace tokens. The backend must securely store these tokens and use them for the specific user's generation tasks to offset platform costs.
- **Notification Preferences:** Subscribing to specific system alters (Budget warnings, Performance drops, Scale opportunities).
